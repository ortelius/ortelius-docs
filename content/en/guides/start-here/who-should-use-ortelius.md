Ortelius is aimed at teams who need to know not just *that* a vulnerability exists, but *where it's actually running* and *how urgently it needs to be fixed*.

## Security teams

Ortelius gives security teams a single, continuously updated view of open vulnerabilities across every deployed release, with SLA tracking (Critical/High/Medium/Low targets) and MTTR metrics so you can report on remediation progress rather than just detection.

## Platform and infrastructure teams

If you run Kubernetes clusters, GKE, or other production environments, Ortelius' post-deployment tracking tells you what's actually live right now — not what was deployed last quarter — by syncing endpoint and release data automatically.

## Engineering and DevOps teams

Connect GitHub or GitLab and Ortelius maps releases, SBOMs, and CVEs back to the repository and commit that introduced them, so engineers can go from "a CVE was disclosed" to "here's the PR that fixes it" quickly.

## Compliance and audit teams

Dashboard metrics are designed to support reporting against common frameworks, including NIST SP 800-53, SP 800-137, SP 800-190, SP 800-218 (SSDF), and Executive Order 14028.

## Open source contributors

Ortelius is itself an open-source project under the Continuous Delivery Foundation. If you'd rather build and extend the platform than adopt it as a user, see the [Developer Resources](../../developer-resources/) section, starting with the [Contributor Guide](../../developer-resources/contributor-guide/).
